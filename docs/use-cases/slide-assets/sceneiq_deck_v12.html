System architecture

Verified contextual prompts, built from grounded scene intelligence

SceneIQ converts Tubi Moments VLM output into source-backed prompts, validates the evidence, ships a client-ready bundle, and turns interaction into high-value learning signals.

1

Input

Tubi Moments VLM JSON: cuepoints, scenes, cast, dialogue, objects, locations, themes, music, and wiki context.

2

Scene Context

Classify domain, gate unsafe or short scenes, and prepare eligible moments.

3

Prompt Generation

Five parallel pipelines generate prompts from sceneInfo, cast enrichment, trivia, realism checks, and editorial facts.

sceneInfo Cast Trivia Realism Facts
4

Trust Layer

Validate URLs, evidence, sources, MCQ quality, dedup, score, and re-fetch cited pages.

5

Client + Learning

Emit UI bundle and audit files, then use engagement signals for model improvement.

Feedback loop: interactions improve prompt ranking, recommendations, user profiles, and ads/partner intelligence.

Validation happens during generation

Prompt candidates must pass URL provenance, source quality, verbatim evidence, cross-source, expert-name, and MCQ option checks before final scoring.

Finalizer consolidates candidates

Dedup across pipelines, score accuracy/legal/UX/monetization dimensions, then route to approve, reject, or needs-edit.

Eval re-checks evidence

Before output, cited URLs are re-fetched, title grounding is verified, and quotes are re-matched against live source text.

Output becomes a learning surface

UI prompts create interaction data that can improve recommendation ranking, prompt ranking, profile quality, and partner intelligence.
